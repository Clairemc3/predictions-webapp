# Feature: AI-Generated Question Short Title

## Status
Implemented

## Summary

When a host creates a new question or updates an existing question's title, the application automatically generates a short title for that question using an AI agent (Google Gemini). The short title is displayed to members when they view their predictions and points. Hosts cannot manually set or edit the short title — it is always AI-generated.

---

## Background & Problem

Questions in the predictions game can have long descriptive titles (e.g. "Which team will finish in the top 4 of the Premier League?"). When members view their answer cards, space is at a premium. Each answer card on a ranking question shows a position number on the left; each answer card on an entity selection question shows a short label on the left.

---

## Requirements

### Functional

| # | Requirement |
|---|---|
| F1 | When a question is created with a title, a short title is automatically generated by an AI agent. |
| F2 | When a question's title is updated, a new short title is generated for the new title. |
| F3 | If a question's title is unchanged on update, no new short title is generated. |
| F4 | A host must never be able to manually enter or edit the short title via any form. |
| F5 | If the same title text has been generated before (across any question), the cached result is reused — no new AI call is made. |
| F6 | Short title generation must not block or slow down the question save action. |

### Short Title Format Rules

The AI agent is instructed to produce a short title that:
- Contains a **maximum of 4 words**
- Has each word at a **maximum of 7 characters** (abbreviate longer words where needed)
- Contains no punctuation or surrounding quotes
- Is plain text only

Examples are provided in the agent's system instructions to guide abbreviation style (e.g. `Top goalscorer` → `Top G/S`).

---

## User Experience

### Host (creating/editing a question)
- The host fills in the question title as normal.
- There is **no short title field** in the form — it has been removed.
- After saving, short title generation happens silently in the background. It typically completes within a few seconds.

### Member (viewing predictions/points)
- On **ranking** questions: the answer card shows the member's predicted position number on the left.
- On **entity selection** questions: the answer card shows the `short_title` on the left of each answer card, replacing the longer question title.
- If the short title has not yet been generated (e.g. immediately after a question is created, before the job runs), it will appear blank until the background job completes.

## Technical Design

### AI Provider

- **Provider:** Google Gemini (via the Laravel AI SDK)
- **Model:** Auto-selected cheapest Gemini model via `#[UseCheapestModel]` (currently `gemini-2.5-flash-lite`)
- **Free tier:** Fully covered by Gemini's free tier for typical usage volumes. Note: prompts are used to improve Google's products on the free tier.

### Caching Strategy

Generated short titles are stored in a dedicated `question_title_caches` database table, keyed by the full title text. This table serves as the persistent cache — since the production database is cloud-hosted, no external cache service is required.

**Cache lookup flow:**
1. Job fires with the question's title text.
2. Check `question_title_caches` for an existing entry with that title.
3. **Cache hit:** apply the stored `short_title` directly to the question — no AI call.
4. **Cache miss:** call the AI agent, store the result in `question_title_caches`, apply to the question.

This means **each unique title is only ever processed once**, regardless of how many questions share the same title.

### Key Components

| Component | Location | Purpose |
|---|---|---|
| `ShortTitleGenerator` | `app/Ai/Agents/ShortTitleGenerator.php` | Laravel AI SDK agent with Gemini config and system instructions |
| `GenerateQuestionShortTitle` | `app/Jobs/GenerateQuestionShortTitle.php` | Queued job — checks cache, calls agent if needed, saves result |
| `QuestionTitleCache` | `app/Models/QuestionTitleCache.php` | Eloquent model for the `question_title_caches` table |
| `QuestionObserver` | `app/Observers/QuestionObserver.php` | Dispatches the job on `created` and `updated` events |

### Data Flow

```
Host saves question
      │
      ▼
QuestionObserver::created() / updated()
      │  (only if title is set / changed)
      ▼
GenerateQuestionShortTitle job dispatched to queue
      │
      ├─── Cache hit? ──► Apply cached short_title to question
      │
      └─── Cache miss?
                │
                ▼
         ShortTitleGenerator (Gemini AI)
                │
                ▼
         Store result in question_title_caches
                │
                ▼
         Apply short_title to question
```

### API Resource

`PredictionQuestionsResource` exposes `short_title` alongside the question's other fields to the frontend.

---

## Out of Scope

- Hosts cannot trigger regeneration of a short title manually.
- There is no admin UI to view or edit the `question_title_caches` table.
- Short title generation is not supported for questions without a title.
- The short title is not manually editable via any API endpoint.

---

## Testing

- **Job tests** (`tests/Feature/Jobs/GenerateQuestionShortTitleTest.php`) cover: null title guard, cache hit path (no AI call), cache miss path (agent faked + result stored), and cache reuse across multiple questions with the same title.
- **Observer tests** (`tests/Feature/QuestionShortTitleObserverTest.php`) confirm: job dispatched on question creation with title, NOT dispatched on creation without title, dispatched on title change, NOT dispatched when updating without changing the title.

---

## Configuration

Add `GEMINI_API_KEY` to `.env`. A free key can be obtained from [Google AI Studio](https://aistudio.google.com/apikey).
